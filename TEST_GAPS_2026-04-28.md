# Test Coverage Gaps — secure-review v0.5.11

## High-value gaps (would catch real regressions)

### Function/path: `src/cli.ts:main()` command wiring and GitHub Actions shim
**Current state:** none
**Gap:** The CLI entrypoint has no smoke tests for `review`, `fix`, `pr`, `scan`, `setup-secrets`, output path selection, exit code mapping, invalid `--max-cost-usd`, or the GitHub Actions no-subcommand shim at `src/cli.ts:362`. Most user-facing regressions would happen here even if unit-tested internals still pass.
**Proposed test:** Add a CLI smoke test that runs the built or tsx-loaded CLI in a temp project with mocked mode modules. Assert `review ./src` writes both configured report/evidence files, `fix ./src --max-cost-usd 0` exits 1 with "Invalid max cost", and a gate-blocked fix exits 2 after writing the report/json/diff. Add one GitHub Actions case with `GITHUB_ACTIONS=true`, no subcommand, `INPUT_CONFIG=custom.yml`, and a minimal pull_request event, then assert it dispatches the `pr` path rather than printing help.

### Function/path: `src/cli.ts:pr` end-to-end PR action flow
**Current state:** none for the CLI path; `postPrReview` has one narrow unit test
**Gap:** The PR command branches for missing `GITHUB_EVENT_PATH`, non-PR events, forked PR skip, missing owner/repo/token, PR file pagination, commentable-line filtering, and critical-on-diff exit are not exercised through the actual action entrypoint.
**Proposed test:** Use a temp event JSON and mocks for `@octokit/rest`, `listPullRequestFiles`, `runReviewMode`, and `postPrReview`. Feed one critical finding on a diff line and assert the process exits 2 only when `block_on_new_critical` is true; feed a fork event and assert `runReviewMode` is not called. This would catch regressions that make the GitHub Action fail open or fail closed.

### Function/path: `src/sast/index.ts:runAllSast()`, `src/sast/semgrep.ts:runSemgrep()`, `src/sast/eslint.ts:runEslint()`, `src/sast/npm-audit.ts:runNpmAudit()`
**Current state:** none; mode tests mock `runAllSast`
**Gap:** SAST is a major input to review/fix, but no tests cover tool selection, disabled mode, path normalization, exit-code semantics, JSON parsing, severity mapping, metadata flattening, or graceful unavailable/error behavior.
**Proposed test:** Mock `node:child_process.spawn` for each runner. Assert Semgrep exit 1 with JSON results returns `available: true`, maps `ERROR` to `HIGH`, flattens `metadata.cwe` arrays, and preserves the raw payload; assert ESLint exit 1 returns findings with severity 2 as `MEDIUM`; assert npm audit exit 1 parses vulnerabilities into `package.json` findings and maps `critical/high/moderate/low`. For `runAllSast`, mock all three runners and assert `enabled: false` calls none, `tools: ['eslint']` calls only ESLint, and absolute SAST paths are normalized relative to the scan root.

### Function/path: `src/adapters/*:complete()`
**Current state:** factory selection only
**Gap:** The concrete OpenAI, Anthropic, Google API, Claude CLI, and Gemini CLI adapters have no tests for request shape, JSON-mode behavior, token/cost extraction, retry wrapping, CLI stdout parsing, stdin handling, or non-zero CLI exit errors.
**Proposed test:** Mock `openai`, `@anthropic-ai/sdk`, and `@google/generative-ai` constructors and assert `complete({ jsonMode: true, maxTokens: 123 })` sends each provider's expected JSON-mode knobs and returns usage/cost from the mocked response. For `AnthropicAPIAdapter`, assert the assistant `{` prefill is sent and prepended back to returned text; for `GoogleAPIAdapter`, assert `responseMimeType: 'application/json'` and optional `GOOGLE_BASE_URL` are passed. Mock `spawn` for CLI adapters and assert Claude writes the prompt to stdin and parses `total_cost_usd`, while Gemini passes the combined prompt as a `-p` argument and rejects on non-zero exit.

### Function/path: `src/modes/fix.ts:runFixMode()`
**Current state:** shallow gate-timing tests with empty findings and heavy mocks
**Gap:** The core fix loop is not tested with real loop semantics: initial union findings, writer invocation, reviewer rotation, `parallel_aggregate` behavior, full-rotation-clean early exit, final verification mode choices, SAST before/after summaries, `filesChanged` collection, and introduced-critical gate handling inside an iteration.
**Proposed test:** Mock reviewer outputs by reviewer name and call sequence: initial scan returns one finding, writer returns one changed file, verifier A returns clean, verifier B returns clean, with two reviewers and `max_iterations: 3`. Assert the writer received the aggregated initial finding, iterations rotate A then B, `filesChanged` includes the writer file, and the loop exits only after two consecutive clean verifier runs. Add a second test where the verifier introduces a new CRITICAL finding and assert `gateBlocked`, merged `gateReasons`, and `finalFindings` reflect the verifier output.

### Function/path: `src/modes/review.ts:runReviewMode()`
**Current state:** one sequential-concurrency test; no behavioral aggregation test
**Gap:** Parallel review, SAST injection into reviewer prompts, SAST disabled/enabled branches, path normalization, aggregation of AI + SAST findings, and total cost computation are not asserted. A regression could drop SAST findings or fail to pass prior findings to reviewers without breaking existing tests.
**Proposed test:** Mock `runAllSast` to return one absolute-path SAST finding and mock two reviewers to return overlapping findings with costs. With `inject_into_reviewer_context: true`, assert each reviewer receives prior SAST findings, output findings are aggregated into one normalized relative path, and total cost sums only reviewer costs. Repeat with `inject_into_reviewer_context: false` and assert prior findings are omitted.

### Function/path: `src/reporters/github-pr.ts:postPrReview()`
**Current state:** one path-normalization test
**Gap:** Important branches are untested: anchoring on the first commentable line inside a finding range, moving touched-file findings outside diff hunks into the summary, dropping untouched-file findings, omitting `comments` when there are no inline comments, rendering remediation/tags/confidence, and returning `criticalOnDiff`.
**Proposed test:** Build a `ReviewModeOutput` with four findings: one starting on a diff line, one whose `lineStart` is outside the diff but `lineEnd` crosses a commentable line, one in a touched file outside all hunks, and one in an untouched file. Assert `createReview` receives two inline comments, the second uses the later anchor line, the summary contains the pre-existing touched-file finding and dropped count, and `criticalOnDiff` counts only inline criticals.

### Function/path: `src/roles/writer.ts:runWriter()`
**Current state:** sanitization, clean write, path traversal, and escaping symlink tests
**Gap:** The retry path added for unparseable writer output is not tested, nor are adapter failure handling, usage accumulation across retry, empty-finding short circuit, broken symlink rejection, and malformed `changes` entries. These are realistic failure modes for model output and filesystem safety.
**Proposed test:** Use a mock adapter that first returns prose and then valid JSON; assert `complete` is called twice, the second prompt contains the stricter JSON-only reminder, usage is summed, and the file is written. Add an error test where the second response is still unparseable and assert `error` is set with the last raw text and no writes occur. Add a temp broken symlink path and assert the error contains "broken symlink".

### Function/path: `src/util/files.ts:readSourceTree()` and `src/util/files.ts:serializeCodeContext()`
**Current state:** only indirect path normalization coverage
**Gap:** The source ingestion logic has no direct tests for single-file roots, ignored directories, hidden files versus `.env.example`, lockfile skipping, extension filtering, max-size skipping, unreadable-file tolerance, stable relative paths, line numbering, and prompt truncation. This code determines what reviewers actually see.
**Proposed test:** Create a temp tree with `src/app.ts`, `.env.example`, `.env`, `node_modules/skip.ts`, `package-lock.json`, `README.md`, `Dockerfile`, and an oversized JS file. Assert only the intended code/config files are returned with correct `relPath` and line counts. For `serializeCodeContext`, pass two files and a small `maxChars`; assert line numbers are rendered and the truncation marker reports the remaining file count.

### Function/path: `src/commands/setup-secrets.ts:runSetupSecrets()`
**Current state:** none
**Gap:** The command shells out to `gh`, parses config directly, filters placeholders, writes secrets through stdin, and exits with different codes, but none of that is tested. Regressions here can leak secrets on argv, set placeholder values, or skip real provider keys.
**Proposed test:** Mock `spawn`, create a temp `.secure-review.yml` with writer/reviewer providers, and stub `process.env` keys including one placeholder. Assert `gh --version` and `gh auth status` are checked, `gh secret set OPENAI_API_KEY --repo owner/repo` receives the value on stdin and not in args, placeholder keys are skipped, unknown providers warn, and a failed `gh secret set` exits 2.

## Lower-value gaps (nice to have, not urgent)

### Function/path: `src/config/load.ts:loadEnv()`, `resolveSkillPath()`, `loadSkill()`
**Current state:** `loadConfig()` is tested; these helpers are only covered indirectly
**Gap:** Invalid environment modes, relative versus absolute skill path resolution, and `loadSkill()` read errors are not directly asserted.
**Proposed test:** Assert `loadEnv({ OPENAI_MODE: 'bad' })` throws `Invalid environment`, `resolveSkillPath('skills/a.md', '/repo')` returns `/repo/skills/a.md`, absolute paths are unchanged, and `loadSkill('/missing')` includes the failed path in the error.

### Function/path: `src/findings/aggregate.ts:mergeFindings()` branches
**Current state:** good basic dedupe coverage
**Gap:** Tests do not assert longest description/remediation preference, missing-CWE title-prefix bucketing, `cwe`/`owaspCategory` fallback from the second finding, or confidence capping at 1 with four reporters.
**Proposed test:** Aggregate three or four overlapping findings without CWE but with matching title prefixes, varied severities, descriptions, remediation, tags, and reporters. Assert the highest severity, longest text, fallback tags, de-duplicated reporter set, and confidence `1`.

### Function/path: `src/findings/parse.ts:parseFindings()` branch variants
**Current state:** strong parser coverage for common JSON forms and salvage behavior
**Gap:** `line_start`/`line_end`, camelCase `lineStart`/`lineEnd`, `fix` as remediation, `owasp_category`/`owasp`, `NOTE` severity, unknown severity defaulting to `MEDIUM`, and a malformed item that passes the raw schema but fails `FindingSchema` are untested.
**Proposed test:** Feed one payload using snake_case fields and `fix`, one using camelCase fields and `owasp_category`, and one with negative `line_start`. Assert normalized line fields/remediation/tags on the valid findings and that the negative-line item is skipped with a warning.

### Function/path: `src/findings/aggregate.ts:countBySeverity()`
**Current state:** none
**Gap:** Exported from the module but unused and untested.
**Proposed test:** Either add a small direct test for `countBySeverity([CRITICAL, HIGH, HIGH], 'HIGH') === 2` or remove the export if it is not part of the intended public API.

### Function/path: `src/gates/evaluate.ts:evaluateGates()`
**Current state:** good coverage for proceed, cost, wall time, new critical, and iteration 0
**Gap:** The `block_on_new_high` branch and simultaneous multi-reason output are not asserted.
**Proposed test:** Enable `block_on_new_high`, introduce one HIGH finding in iteration 1, and assert the decision blocks with a `HIGH` reason. Add cost + wall-time + new-critical together and assert all expected reasons are preserved.

### Function/path: `src/util/diff.ts:commentableLinesFromPatch()`
**Current state:** solid hunk parsing coverage
**Gap:** The `\ No newline at end of file` marker, unknown-prefix fallback, hunk headers without explicit lengths, and file header lines inside patches are not tested.
**Proposed test:** Feed a patch with `@@ -3 +3 @@`, `+++ b/file`, `--- a/file`, a `\ No newline at end of file` marker, and one unknown prefix. Assert only real new-side lines are added and the unknown line advances as context.

### Function/path: `src/reporters/json.ts:renderFixEvidence()`
**Current state:** basic evidence shape tested
**Gap:** No test covers introduced findings, zero-initial-findings resolution rate, gate-blocked notes, `sourceCondition`/`sessionId`, rounded per-iteration cost, or semgrep/eslint counts from the last iteration.
**Proposed test:** Build a fix output with no initial findings and one final introduced finding; assert `resolution_rate_pct` is 0 and `new_findings_introduced` is 1. Build a gate-blocked one-iteration output and assert `notes`, last `sastAfter` counts, and rounded `per_iteration.cost_usd`.

### Function/path: `src/commands/init.ts:runInit()`
**Current state:** generators are well tested; filesystem orchestration is not
**Gap:** Refuse-to-overwrite, `--force`, appending to existing `.env.example`, unchanged marker behavior, active/example workflow path creation, and "no providers" / disabled writer provider exits are not tested.
**Proposed test:** Run `runInit({ yes: true })` in a temp cwd and assert `.secure-review.yml`, `.env.example`, and `.github/workflows/secure-review.yml.example` are created. Add an existing config case that stubs `process.exit` and asserts no overwrite without `force`; add an existing env file with the marker and assert it remains unchanged.

### Function/path: `src/util/logger.ts` and `src/util/spinner.ts`
**Current state:** none
**Gap:** Quiet/verbose behavior and non-TTY spinner output are not tested. This is mostly UX, but regressions can make CI logs noisy or silent.
**Proposed test:** Spy on `console.log/warn/error` and `process.stderr.write`; assert `setQuiet(true)` suppresses `info`, `success`, and spinner output but still lets warnings/errors through, and `setVerbose(true)` enables debug output.

## Coverage that's already strong

- `src/config/schema.ts`: meaningful schema defaults/rejections are covered for findings, evidence JSON, secure-review config, and env modes.
- `src/config/load.ts:loadConfig()`: valid YAML, defaults, config directory, and invalid config errors are covered.
- `src/findings/parse.ts:extractJson()`: bare JSON, fenced JSON, prose-wrapped JSON, no-JSON errors, and jsonrepair fallback for single quotes, trailing commas, truncated output, and prose suffix are covered.
- `src/findings/aggregate.ts:aggregate()` and `src/findings/diff.ts:diffFindings()`: core dedupe by file/CWE/line bucket, ID renumbering, severity escalation, reporter union, path-normalized merge, severity breakdown, and resolved/remaining/introduced diffing are covered.
- `src/gates/evaluate.ts`: the main gate triggers are covered for cost, wall time, new criticals, and the iteration-0 exception.
- `src/util/retry.ts`: transient status/code/message detection, nested `cause` handling through production code, retry success, non-transient no-retry, and max-attempt exhaustion are mostly covered.
- `src/util/cost.ts`: known-model cost, fallback pricing, and linear scaling are covered.
- `src/adapters/factory.ts`: Anthropic API/CLI selection, CI CLI refusal, OpenAI CLI refusal, and missing API-key errors are covered.
- `src/roles/reviewer.ts`: structured response parsing, adapter/parser failure capture, malformed-item salvage, reviewer attribution, and usage propagation are covered with a mock adapter.
- `src/roles/writer.ts`: NUL/control-character sanitization, clean-content writes, path traversal refusal, and symlink escape refusal are covered.
- `src/util/diff.ts` and `src/util/github-pr-files.ts`: common PR patch line parsing and Octokit pagination are covered.
- `src/reporters/markdown.ts` and basic `src/reporters/json.ts`: review/fix report rendering and Condition-D-compatible evidence basics are covered.
- `src/commands/init.ts` generator functions: generated config, env, workflow, provider omissions, custom writer model, SAST toggle, max-iteration clamp, and workflow permissions are covered.
